6.10
1.重构红外加权逻辑：从原始加权累加改为"邻居确认+同侧确认"策略，以应对比赛场地严重的红外反射问题

  问题背景：
  - 红外传感器用于检测光电门方向（非避障），传感器呈圆环排列：5正前、2正右、8正左
  - 比赛场地水面/墙壁反射严重，导致对侧传感器被误触发
  - 原方案所有触发传感器权重直接累加，反射导致正负权重相互抵消甚至反转转向方向
  - 例：门在右侧，IRM_3(+3)、IRM_4(+2)被直射触发，IRM_7(-3)、IRM_8(-4)被反射误触发
        权重和 = (+3+2)+(-3-4) = -2 → 船向左转，背离正确的门

  新策略设计思路：
  核心假设：光电门为点状红外源，直射信号触发2~3个相邻传感器形成连续弧段；
           反射信号通常是散乱的孤立触发，无邻居伴随且难以集中在同侧

  确认规则（满足任一即视为真信号）：
  (1) 仅1个传感器触发 → 直接信任（弱信号不丢弃，无对侧抵消风险）
  (2) 物理邻居也触发 → 确认（邻接传感器形成连续弧段，符合点源特征）
  (3) 同侧有≥2个触发 → 确认（不邻接但在同侧，方向一致，如IRM_2和IRM_4同时触发）

  三条都不满足 → 判定为反射，丢弃

  具体改动：
  /1. infrared.h 新增邻居查找表 ir_neighbor_left[] 和 ir_neighbor_right[]
      编码圆环上每个传感器的左右邻居索引（含首尾循环）
  /2. infrared.c 重写 Infrared_CalcWeightSum()
      统计触发总数及左右侧触发数 → 单触发走原始逻辑 → 多触发按三条规则确认
  /3. main.c 无需修改，接口不变

  典型场景推演：
  - IRM_3,4触发(邻接) + IRM_7(孤立反射) → 3,4邻接确认，7丢弃 → 正确右转
  - IRM_2,4触发(同侧不邻接) → 同侧确认 → 正确右转
  - IRM_3(真) + IRM_7(反射) → 两侧各1不够同侧阈值，均无邻居 → 全部丢弃 → 直行（安全默认）
  - 仅IRM_3触发(远距离弱信号) → 触发数=1，直接信任 → 正确右转

2.舵机转向方向修复
  问题：下水测试发现站在船左侧发射红外信号，船向右转（背离信号源）
  原因：舵机物理行为是低角度=右转、高角度=左转，但代码假设相反
       左侧传感器(IRM_6,7,8)触发 → 负权重和 → 角度减小 → 代码以为左转 → 实际右转
  修复：sg90.c中 SG90_CalcTargetAngle 的权重系数从 +5 翻转为 (-5)
       左侧触发 → 负权重×(负系数)=正偏移 → 角度增大 → 物理左转 ✓
       右侧触发 → 正权重×(负系数)=负偏移 → 角度减小 → 物理右转 ✓
  调参：实测-5转向略不足，系数经-6最终调整为-7
